## 3. 수정 완료 (코드 변경)

### V3 해결 — XSS 방어

**파일:** `dashboard.html`

`esc()` 이스케이프 함수를 추가하고, DB에서 가져온 모든 데이터를 HTML에 삽입할 때 적용했다.

수정 범위:
- 수위 테이블 (`r.station`, `r.river`, `r.gu` 등)
- 지도 팝업 (관측소명, 하천명, 수위)
- 생물 카드 (종명, 일반명, 관찰자, photo_url)
- photo_url은 `https://` 프로토콜 검증 추가

### V4 해결 — HTTPS 전환

**파일:** `river_monitor.py:31`

```
변경 전: http://openAPI.seoul.go.kr:8088/...
변경 후: https://openAPI.seoul.go.kr:8088/...
```

서울시 API가 HTTPS를 지원하지 않을 경우 롤백 필요.

### V5 해결 — 중복 방지

**파일:** `species.py`, `river_monitor.py`, `supabase_setup.sql`

- `species.py`: iNaturalist 관찰 ID(`inaturalist_id`)를 저장하도록 수정. Supabase에 upsert 헤더(`resolution=ignore-duplicates`, `On-Conflict`) 추가.
- `river_monitor.py`: `station + measured_at` 조합으로 중복 방지 헤더 추가.
- SQL: `species_observations`에 `inaturalist_id` unique index, `river_readings`에 `(station, measured_at)` unique constraint 준비.

### V6 해결 — 헬스체크

**파일:** `species.py`, `river_monitor.py`, `dashboard.html`, `supabase_setup.sql`

- 각 수집기에 `report_health()` 함수 추가 — 수집 성공/실패 시 `collector_health` 테이블에 기록.
- API 호출 실패 시 에러 메시지도 함께 저장.
- 대시보드 헤더에 마지막 수집 시각 + 상태(ok/error) 표시 추가.
- SQL: `collector_health` 테이블 생성 스크립트 준비.

## 4. 추가 버그 수정

### 하천명 공백 중복 문제

**증상:** 보고서와 EHI 패널에서 "도림천"과 "도림천(공백)"이 별개 항목으로 표시됨.

**원인:** 서울시 API가 반환하는 하천명에 trailing whitespace가 포함되어 있었음.

**수정:**
- `river_monitor.py`: `.strip()` 적용 (향후 데이터)
- `dashboard.html`: EHI 로딩 시 `trim()` 처리 (기존 데이터 표시)
- `supabase_setup.sql`: 기존 데이터 정리 UPDATE 쿼리 추가

### 대시보드 자동 새로고침

`setInterval`에 `loadEHI()`가 빠져 있어 EHI 패널만 갱신되지 않던 문제 수정.

## 5. 사용자 실행 필요 사항 (Supabase)

아래 SQL은 Supabase 대시보드 > SQL Editor에서 직접 실행해야 한다.

### 5-1. 보안 강화 (V1+V2)

anon key의 INSERT 권한을 제거하고, service_role 전용 INSERT 정책을 생성한다.

```sql
-- anon INSERT 정책 제거
drop policy if exists "river_readings_insert" on river_readings;
drop policy if exists "species_insert" on species_observations;
drop policy if exists "alerts_insert" on invasive_alerts;
drop policy if exists "ehi_insert" on ehi_scores;

-- service_role 전용 INSERT 정책
create policy "service_insert_readings" on river_readings
  for insert to service_role with check (true);
create policy "service_insert_species" on species_observations
  for insert to service_role with check (true);
create policy "service_insert_alerts" on invasive_alerts
  for insert to service_role with check (true);
create policy "service_insert_ehi" on ehi_scores
  for insert to service_role with check (true);
```

### 5-2. 중복 방지 + 헬스체크 테이블 (V5+V6)

```sql
-- 하천명 공백 정리
update river_readings set station=trim(station), river=trim(river), gu=trim(gu)
where station<>trim(station) or river<>trim(river) or gu<>trim(gu);

update ehi_scores set river=trim(river) where river<>trim(river);

-- species 중복 방지
alter table species_observations add column if not exists inaturalist_id bigint;
create unique index if not exists uq_species_inat_id
  on species_observations(inaturalist_id) where inaturalist_id is not null;

-- readings 중복 방지
alter table river_readings
  add constraint uq_readings_station_time unique (station, measured_at);

-- 헬스체크 테이블
create table if not exists collector_health (
  id bigint generated always as identity primary key,
  collector text not null,
  status text not null default 'ok',
  record_count int default 0,
  error_message text,
  collected_at timestamptz default now()
);
alter table collector_health enable row level security;
create policy "health_read" on collector_health for select using (true);
create policy "service_insert_health" on collector_health
  for insert to service_role with check (true);
```

### 5-3. service_role key 설정

1. Supabase > Settings > API > `service_role` key 복사
2. `.env` 파일에 추가: `SUPABASE_SERVICE_KEY=복사한키`
3. 완료 후 Python 코드에서 키 전환 (다음 세션에서 진행)
